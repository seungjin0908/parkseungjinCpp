## DLL 이란?

```
라이브러리를 갖다 쓰잖아요?


라이브러리의 생김새
 - rapidjson 
  => *.h 파일과, *.hpp 파일로 이루어진 라이브러리
  => 소스코드 뭉치였어요.

 - EntityFramework
  => dll 을 줍니다.

 여기서 그럼 과연 DLL이 무엇인가?
```

### DLL의 역할
```
 우리가 코드를 적은걸 그대로 노출해버리면, 
 우리 자산이 남들에게 뿌려지는 거잖아요.

 회사대 회사로 외주계약을 했습니다.
 A회사가 B회사한테 자동주행 시스템 프로그램을 만들어서 라이센스 계약을 했어요.
 한달에 한번씩 돈을 내게끔 계약이 됐습니다.

 소스코드를 줘버리면, 다음 계약이 없겠죠.

 소스코드를 공개안하는 선에서
 우리가 만든 함수를 사용할수있게 해주는 방법이 DLL을 전달하는 방법입니다.

 DLL은 바이너리로 이루어져있기때문에,
 어떤함수가 있는지만 알수있고 실제 함수코드는 볼수없습니다.
```
