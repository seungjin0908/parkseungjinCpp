### 테이블 관계

``` 
 1:1 관계
  - 잘 안쓰입니다.

 1:N 관계 (일대다)
  - 1개의 A 테이블이 N개의 B 테이블을 가지고있는 경우.
  (쿠팡으로 따지면)
   한명의 유저(A테이블)
     - N개의 상품(B테이블) 을 구매
  
 N:N 관계 (다대다)
  - 페이스북 인스타그램 태그 시스템
  
  콘서트 다녀왔다.
  #BTS #콘서트 ...
  1개의 게시글이 N개의 태그를 가지고있죠
  1개의 태그가 N개의 게시글을 가지고있기도 합니다.
```



### Foreinkey
```
 ForeinKey는 A 테이블과 B 테이블의 관계를 나타내는 키에요.

 관계를 알기때문에, DB Tool에서는 선을 그어서 확실하게 표현해줄수있겠죠.
```

#### 단점
```
 Foreinkey를 사용하면 불편한점이 생깁니다.
  그래서 => 굳이 표현을 안해도, 사용하는사람끼리 알아서 잘 사용하면 되지않나?

 사용하면 불편한점 
  - 연관되어있는 테이블이 있으면,
    테이블을 삭제하고싶어도, 연관된 테이블을 먼저 삭제해야합니다.

 Create, Update, Delete가 아주조금 느려집니다.
```

#### 장점
```
 Tool에서 한눈에 확인이 가능
 ORM에서 큰 이점을 발휘합니다.


 Index를 걸기때문에, 조회가 빨라집니다.
 포린키로 인덱스를 걸면, 나중에 포린키로 조회를 하기 때문에 조회가 아주 조금 빨라집니다.
```
