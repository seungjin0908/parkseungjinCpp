
## Managed Language (C#) vs  UnManaged Language (C++)
```
 관리되는 언어 (자동으로 메모리관리) vs 관리가 되지않는 언어 (수동으로 메모리관리)
 (메모리)

 
 C++할때는 당연스럽게도 객체를만들면, 객체를 삭제해줬습니다.
 C#에서는 그럴 필요가 없습니다.

 
 [가비지컬렉션] [가비지컬렉터]
  => 쓰레기 수집가
  => 주기적으로 가비지컬렉션이 동작해서 그 어떤 변수도 참조하고있지않은 메모리를 알아서 삭제합니다.

  주기적으로 (랜덤한 타이밍) < 단점
  알아서 삭제합니다. < 장점


  
  단점의 예시.
	  저희가 철권을 만든다고 칩니다.

	  (new, delete) 는 다른 연산자에 비해 30만배 느려요.
	  1프레임정도는 밀릴만한 부하가 맞습니다.

	  철권을 만드는데
	  정권찌르기하는데, 갑자기 렉걸려버리면
	  유저한테 불쾌한 경험을 선사하겟죠.

	  이거는 이제 버그니까 고쳐야지? 했는데
	  언어특성이라 고치질 못합니다. (물론 개선할 여지는 많아요.)

  장점의 예시.
	  C++ 신입개발자가 들어왔습니다.
	  아쉽게도 많이 못해요.
	  그래도 코드리뷰를 하긴했지만 놓치는 부분이 생기겠죠.

	  예를들어 new를 해줬는데, delete를 그 어디에서도 안해요.
	  그래도 코드는 잘 동작하죠.
	  넘어가고 그런 코드가 쌓이면

	  메모리 누수가 생깁니다.

	  ※ 메모리 누수란?
	   삭제되지 않은 메모리가 계속 쌓이는거에요.
	   그래서 프로그램을 켜놓고 오래켜놓으면 메모리가 엄청 올라가있는
	   실제로 사용하는 양보다.

```
